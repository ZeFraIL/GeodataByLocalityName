# Техническое описание класса GeoActivity

Данный документ содержит подробный разбор основного класса приложения GeodataByLocalityName.

---

## 1. Общая информация
*   **Имя класса:** `GeoActivity`
*   **Тип:** Activity (Экран приложения)
*   **Назначение:** Является главным и единственным экраном приложения. Класс отвечает за взаимодействие с пользователем (ввод данных), получение географических координат через интернет и отображение результата.
*   **Взаимодействие:** Напрямую работает с разметкой интерфейса (`activity_geo.xml`), системным сервисом `Geocoder` и внутренним классом `NetworkReceiver`.

---

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `etCityName` | `EditText` | Поле для ввода названия города. | `onCreate`, `findCoordinates` |
| `etCountryName` | `EditText` | Поле для ввода названия страны. | `onCreate`, `findCoordinates` |
| `btnFindCoordinates` | `Button` | Кнопка запуска поиска. | `onCreate`, `NetworkReceiver` |
| `tvResult` | `TextView` | Текстовое поле для вывода результата или ошибок. | По всему классу |
| `networkReceiver` | `NetworkReceiver` | Объект для отслеживания состояния интернета. | `onCreate`, `onDestroy` |
| `progressBar` | `ProgressBar` | Индикатор загрузки (вращающееся кольцо). | `onCreate`, `findCoordinates` |

---

## 3. Методы класса

### Метод: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:** `Bundle savedInstanceState` (сохраненное состояние экрана).
*   **Что делает:** Инициализирует Activity. Связывает переменные с элементами интерфейса, устанавливает слушатель нажатия на кнопку и регистрирует приемник состояния сети (`NetworkReceiver`).
*   **Когда вызывается:** При самом первом запуске экрана системой Android.

### Метод: `onDestroy`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет.
*   **Что делает:** Выполняет "уборку" ресурсов. В данном случае отменяет регистрацию `networkReceiver`, чтобы приложение не тратило ресурсы впустую после закрытия.
*   **Когда вызывается:** Перед окончательным закрытием Activity.

### Метод: `findCoordinates`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `cityName` | `String` | Текст с названием города. |
    | `countryName` | `String` | Текст с названием страны. |
*   **Что делает:**
    1. Создает объект `Geocoder` для работы с картами Google.
    2. Запускает новый поток (`Thread`), так как сетевые операции запрещены в главном потоке.
    3. Пытается найти координаты по адресу через `geocoder.getFromLocationName`.
    4. Возвращается в главный поток через `runOnUiThread`, чтобы обновить текст на экране и скрыть `ProgressBar`.
*   **Когда вызывается:** При нажатии на кнопку "Find coordinates" (после проверки ввода).
*   **Важно понимать:** Если интернет медленный или сервер Google недоступен, может возникнуть ошибка `IOException`, которая обрабатывается блоком `catch`.

---

## 4. Жизненный цикл (Activity)
*   **`onCreate()`**: Вызывается при создании. Здесь мы "строим" экран и готовим данные.
*   **`onDestroy()`**: Вызывается при уничтожении. Здесь мы отключаем функции слежения за сетью, чтобы избежать утечек памяти.

---

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `EditText` (ввод), `Button` (действие), `TextView` (вывод), `ProgressBar` (индикация).
*   **Связь:** Используется метод `findViewById(R.id...)` для поиска элементов, определенных в XML-файле.
*   **События:** Обрабатывается нажатие кнопки через `setOnClickListener`.

---

## 6. Внутренний класс: NetworkReceiver
Этот класс находится внутри `GeoActivity` и является **BroadcastReceiver** (приемником широковещательных сообщений).

*   **Назначение:** Следит за тем, включен ли интернет на устройстве.
*   **Метод `onReceive`**: Вызывается системой Android каждый раз, когда состояние сети меняется (например, выключили Wi-Fi).
*   **Логика:** Если интернета нет, метод отключает кнопку поиска (`setEnabled(false)`) и пишет предупреждение. Если интернет появился — включает кнопку обратно.

---

## 7. Общая логика класса
1. Приложение запускается, проверяет интернет.
2. Пользователь вводит данные города и страны.
3. Нажимает кнопку.
4. В фоновом потоке приложение "спрашивает" у сервиса Google координаты.
5. Результат (широта и долгота) выводится на экран.

---

## 8. Упрощенное объяснение (простыми словами)
Представь, что `GeoActivity` — это **диспетчер в справочном бюро**.
1. У диспетчера есть **бланки** (`EditText`) для записи вопроса.
2. Когда вы нажимаете кнопку, диспетчер не сам ищет ответ (чтобы не задерживать очередь), а отправляет **посыльного** (новый поток `Thread`) в огромную библиотеку (сервер Google).
3. Пока посыльный бегает, на столе крутится **песочные часы** (`ProgressBar`).
4. Когда посыльный возвращается, диспетчер пишет ответ на **табло** (`TextView`).
5. Также у диспетчера есть **радиоприемник** (`NetworkReceiver`), который сообщает, не оборвалась ли связь с внешним миром. Если связь оборвалась, диспетчер вешает табличку "Закрыто" на кнопку.

## ⚠️ Рекомендации по улучшению
*   **Bugs/Bad Practices:** Использование `new Thread(...).start()` считается устаревшим подходом. В современном Android рекомендуется использовать `Coroutines` (для Kotlin) или `Executors` (для Java).
*   **Улучшение:** Добавить проверку на наличие Google Services, так как без них `Geocoder` может не работать.
