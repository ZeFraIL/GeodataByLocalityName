🎯 Цель проекта
• **Какую задачу решает приложение:**  
Позволяет пользователю получить географические координаты (широту и долготу) любого населенного пункта по его названию и стране.

• **Почему эта задача важна:**  
Это базовый инструмент для путешественников, логистов и разработчиков, которым необходимо быстро узнать точное местоположение объекта без использования сложных картографических сервисов.

• **Целевая аудитория:**  
Обычные пользователи, которым нужны координаты для GPS-навигаторов или справочных целей.
________________________________________
📌 Требования к приложению
**Функциональные требования**
• Ввод названия города и страны.
• Получение координат через Google Geocoder API.
• Отображение прогресса выполнения запроса (ProgressBar).
• Мониторинг состояния сети в реальном времени.

**Нефункциональные требования**
• **Производительность:** Выполнение сетевых запросов в фоновом потоке для предотвращения зависаний интерфейса.
• **Удобство использования:** Минималистичный интерфейс, интуитивно понятные поля ввода.
• **Надёжность:** Обработка ошибок отсутствия интернета и ситуаций, когда адрес не найден.

🧠 Общая архитектура
• **Выбранный подход:**  
MVC (Model-View-Controller) на базе одной Activity.

• **Почему выбран именно он:**  
Для небольшого утилитарного приложения такой подход обеспечивает минимальный объем кода и высокую скорость разработки при сохранении читаемости.

• **Основные компоненты системы:**  
- `GeoActivity`: Главный контроллер и экран.
- `Geocoder`: Системный сервис для прямого геокодирования.
- `NetworkReceiver`: Компонент для отслеживания состояния интернета.

- **Как это помогает масштабированию:** В дальнейшем можно легко заменить `Geocoder` на Retrofit-запрос к внешнему API или вынести логику в `ViewModel` (MVVM).

🧩 Подробное описание классов
📌 **Класс: GeoActivity**
**Роль:**  
Основная точка входа и единственный экран приложения.

**Ответственность:**  
Управление жизненным циклом, обработка пользовательского ввода, обновление UI.

**Основные методы:**  
- `onCreate()` — инициализация UI и регистрация ресивера сети.
- `findCoordinates()` — запуск фонового потока для обращения к Geocoder.
- `onDestroy()` — освобождение ресурсов и отмена регистрации ресивера.

**Взаимодействие с другими классами:**  
Взаимодействует с `NetworkReceiver` для блокировки кнопки поиска при отсутствии сети.
________________________________________
🔄 Диаграмма работы приложения
1. Пользователь вводит "Tel Aviv" и "Israel".
2. Нажимает "Find Coordinates".
3. Приложение проверяет наличие интернета.
4. Запускается фоновый поток.
5. `Geocoder` возвращает список адресов.
6. Первый результат отображается на экране.

🎨 UI/UX анализ
• **Почему интерфейс сделан именно так:**  
Максимальная простота для быстрого получения результата.

• **Какие принципы использованы:**  
- **Простота:** Всего два поля и одна кнопка.
- **Логичность:** Сверху вниз: ввод -> действие -> результат.
- **Доступность:** Кнопка активна только при наличии интернета, что предотвращает бесполезные нажатия.

• **Что можно улучшить:**  
Добавить автодополнение названий городов и кнопку "Показать на карте".

⚙️ Работа с потоками
• **Используются:**  
- `Thread` (для сетевого запроса).
- `runOnUiThread` (для обновления UI).

• **Почему выбран этот способ:**  
Прямое использование `Thread` оправдано в учебных и небольших проектах для понимания основ многопоточности в Android без перегрузки проекта внешними библиотеками.

• **Как предотвращаются:**  
- **Зависания (ANR):** Все тяжелые операции вынесены из главного потока.
- **Утечки памяти:** Ресивер отключается в `onDestroy()`.
________________________________________
💾 Работа с данными
• **Где хранятся данные:**  
Данные не сохраняются локально; они запрашиваются динамически.

• **Почему выбран этот способ:**  
Координаты могут меняться или уточняться, а локальное хранение базы городов нецелесообразно по объему.
________________________________________
🌐 Работа с сетью
• **Как выполняются запросы:**  
Через стандартную библиотеку `android.location.Geocoder`.

• **Обработка ошибок:**  
Используется блок `try-catch` для перехвата `IOException`.

• **Что происходит при отсутствии интернета:**  
`NetworkReceiver` отключает кнопку поиска и выводит предупреждение.
